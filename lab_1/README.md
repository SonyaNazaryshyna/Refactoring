# **Лабораторна робота 1**
## **Аналіз коду: виявлення проблемних місць**

## **КРОК 1**
Для виконання лабораторної роботи було обрано невеликий навчальний проєкт — реалізацію гри Yahtzee мовою програмування Python.

Проєкт містить один основний клас Yahtzee, який реалізує логіку підрахунку очок для різних комбінацій кидків п’яти кубиків.

Проєкт складається з:
- одного класу Yahtzee
- методів для підрахунку простих категорій (ones, twos, threes, fours, fives, sixes)
- методів для підрахунку складних комбінацій (pair, two_pair, three_of_a_kind, four_of_a_kind, fullHouse)
- методів перевірки стрітів (smallStraight, largeStraight)
- методу підрахунку загальної суми (chance)

## **КРОК 2-3: Визначення основних проблем/шкідливі наслідки**
### 1. Дублювання коду (Code Duplication)
```
    counts[d1-1] += 1
    counts[d2-1] += 1
    counts[d3-1] += 1
    counts[d4-1] += 1
    counts[d5-1] += 1
```
або
```
tallies[d1-1] += 1
tallies[d2-1] += 1
```

Одна й та сама логіка підрахунку кількості значень кубиків повторюється в багатьох методах. Це порушує принцип DRY та ускладнює підтримку коду.

### 2. Магічні числа (Magic Numbers)
Проблемні рядки:
```
counts = [0]*6
return 50
return 15
return 20
return (i+1) * 4
```

6 — кількість граней кубика
50 — очки за Yahtzee
15 — small straight
20 — large straight

Числа використовуються без пояснення їх значення, що знижує читабельність та ускладнює зміну правил гри.

### 3. Надмірна складність
Проблемний фрагмент (метод score_pair):
```
for at in range(6):
    if (counts[6-at-1] == 2):
```
Незрозуміла логіка індексації (6-at-1) ускладнює читання коду та підвищує когнітивне навантаження.

### 4. Великий клас 
Клас Yahtzee

- Клас зберігає стан кубиків (self.dice)
- Містить логіку підрахунку всіх комбінацій
- Містить як статичні, так і нестатичні методи

Клас виконує занадто багато обов’язків, що порушує принцип єдиної відповідальності (Single Responsibility Principle).

### 5. Невдалі назви змінних
```
_1, 
_2
s
at
sum
```
Змінні мають неінформативні назви
Використання sum перекриває вбудовану функцію Python
Погіршується читабельність та зрозумілість коду

### 6. Непослідовність архітектури
Частина методів використовує @staticmethod
Частина працює через self.dice
Методи приймають або 5 окремих аргументів, або список

Відсутня єдина стратегія організації коду, що ускладнює його підтримку та розширення.
